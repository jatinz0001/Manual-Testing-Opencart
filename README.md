#  Manual Testing Project – OpenCart E-Commerce (Demo)

## Project Overview

This project demonstrates manual testing of an e-commerce application using the OpenCart demo website (sandbox environment). The goal was to validate core functionalities and identify defects through structured test case design and execution.

---

##  Objectives

* Design and execute manual test cases
* Validate key user workflows
* Identify and report defects
* Document testing results with proper structure

---

##  Application Under Test

* OpenCart Demo Website (Sandbox environment used for testing)
* Note: This is a demo platform used to simulate real-world testing scenarios without affecting real users or transactions.

---

##  Tools & Technologies

* Microsoft Excel (Test Cases, Bug Report, Test Summary)
* Browser: Google Chrome
* GitHub (Project hosting)

##  Test Summary

* Total Test Cases: 20+
* Passed: 17
* Failed: 3

---

##  Bug Reporting

* Bugs are documented in the Excel file with:

  * Bug ID
  * Description
  * Steps to reproduce
  * Expected vs Actual results
  * Severity
  * Screenshot reference/link

* Screenshots are linked within the Excel file for each applicable defect.

---

##  Screenshots

* Screenshots are captured for UI-related issues and linked directly in the Excel document.
* For time-based or non-visual issues (e.g., session timeout), detailed observations are provided instead of screenshots.

---

##  Key Learnings

* Writing effective test cases (positive, negative, edge cases)
* Identifying real-world defects in e-commerce workflows
* Structured bug reporting with proper severity classification
* Importance of test documentation and traceability

---

##  Additional Notes

* The testing was performed on a demo (sandbox) environment, which simulates real-world application behavior.
* Some limitations exist due to the nature of demo environments (e.g., restricted backend validation, limited data persistence).
